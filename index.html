How it works

Set up once.
Publish forever.

One-time GitHub setup, then it's a single click every time. No CLI, no CI pipelines, no config files.

01

Create a GitHub repo

Make a new repository for your blog on GitHub. Enable Pages. Takes two minutes. You only do this once.

02

Paste your access token

Generate a GitHub personal access token and paste it into Thooval. That's the only "technical" step. You control the token.

03

Click Publish every time

Write in the WYSIWYG editor. Hit Publish. Thooval builds clean static HTML and pushes it to GitHub Pages — your free hosting — automatically.

Key Features

A real editor.
Actual ownership.

Thooval is built around a Typora-style WYSIWYG editor with one-click GitHub Pages deployment. Everything else follows from that.

Core

WYSIWYG editor — like Typora

Write and see your post formatted as you type. Bold is bold. Headings look like headings. No raw **markdown** syntax cluttering your view.

Core

One-click GitHub Pages deploy

Hit Publish. Thooval generates clean HTML and pushes it directly to your GitHub Pages repo. Your blog is live — for free — in seconds.

Ownership

Local-first — .md files on your Mac

Every post is a plain .md file on your disk. No cloud storage, no Thooval account. Open them in any editor. They're yours unconditionally.

Privacy

No account required

Download, install, start writing. No sign-up email, no password, no profile. You don't need an account with Thooval — ever.

Analytics

Google Analytics built-in

Paste your GA4 measurement ID once. Thooval embeds the tracking tag in every page automatically. See who's reading — no plugins needed.
